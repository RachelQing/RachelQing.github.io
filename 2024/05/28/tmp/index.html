npm package install error "npm ERR! code CERT_HAS_EXPIRED".

为了求学和求职，想要快递建立一个个人主页。

几年前也曾有过类似想法，想着不能任由自己再这样自闭下去了。但当时有工作压力，社交压力，性格依旧高内耗从而放大了所有负面压力。因此并没有在office time外坚持任何有益工作的活动，包括这一想法也并未能实际展开。今天找回闲置4年的github，才发现14年（近10年前）曾在同事鼓动下新建了个jekyll repo，但内容空无一物，连Github pages设置都没搞好。

Google一下，这几年来主流方向似乎没什么变化，依旧是：WordPress，Google Site，Hexo和Typecho。（后两个并不是新工具这件事，还是从自己的Github的团队项目里瞥见这两的痕迹，才发现的。）由于我实在想既快速又可编码，并不想考虑准备vps的方案，最后决定就是Hexo。

Hexo由Nodejs开发。我虽然对nodejs不熟，但好歹使用没问题，本地环境也都还健在。没想到第一步安装npm包就被磕绊一脚。

上面这个命令报错“npm ERR! code CERT_HAS_EXPIRED”。我脑子没反应过来，第一反应是npm怎么会存在过期的问题呢？大概是我的npm版本太低了（v6的确也相当老旧了）。于是遇事不决，版本升级（其实没用）。

但气恼人的是，npm install npm -g成功也依旧版本未改变（还不如不成功）。奇了怪了，这一下子给我搞蒙了，性子一急就扎进去了。这时候，我脱离工作太久的大脑并没有想到，npm这样的包管理工具都是依附于特定语言包的，npm则依附于nodejs。nodejs版本直接决定了npm版本的范围。我忙活好一阵，才想起来我本地采用nvm在管理nodejs版本。于是我用nvm下载了最新版nodejs。打击人的是，最新版本v21下载成功却不支持我的MacOS版本（又不在下载安装时报错！）。改为较老一些的v16，则可成功使用。nodejs v16对应着npm v8。

然而一波三折，最后发现npm版本更新没用。根本原因不在版本，而在于npm源失效了。啊~~~我脑子实在愚钝，怎么没想到npm作为包管理器肯定得配置国内源才能实用！？

最后，“npm ERR! code CERT_HAS_EXPIRED”根本原因是npm的国内源改hostname域名了。
