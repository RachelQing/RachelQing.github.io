Created2024-01-01|Updated2024-05-31

基础信息

Hexo 是一个基于 Node.js 的静态博客框架，它允许快速创建和部署个人博客或静态网站。其默认使用 Markdown 编写文章，通过命令行工具进行操作（包括将md文件生成为静态页面）。加上利用github的静态网站托管服务，即可方便实现免费的个人主页发布。

简单来说，使用hexo+github发布blog的一般流程是：本地撰写新blog的markdown文件，利用hexo命令行生成静态网页，部署到github。

我的操作环境为MacOS。下面基于这一系列前提进行操作。


准备本地hexo环境

安装nodejs+npm：推荐使用nvm进行nodejs版本管理。

安装hexo的npm包：

 客户端：$ npm install hexo-cli -g

 git自动部署：$ npm install hexo-deployer-git –save

MacOS必要步骤：

 $ brew install hexo

关于Github Pages

参见 官方文档。

GitHub Pages 是 GitHub 提供的免费托管服务，它允许用户将静态网站部署到 GitHub上，并通过自定义域名或 GitHub 提供的域名进行访问。

GitHub Pages 支持多种静态网站生成器，包括 Jekyll 和 Hexo。Jekyll 是 GitHubPages 的默认静态网站生成器，Hexo则并没有与 GitHub Pages 的默认集成。因此Github支持Jekyll项目在线构建，并要求其将静态网页推送到 gh-pages 分支。

GitHub Pages提供站点地址xxx.github.io，xxx根据repo类型不同（用户，项目，组织）存在定名差异。就个人用户而言，简单来说每个github个人用户只有唯一的.github.io。

若有资源采用自定义域名，可参考关于自定义域名和 GitHub 页面。
