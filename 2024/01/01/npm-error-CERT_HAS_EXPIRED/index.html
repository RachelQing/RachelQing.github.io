npm package install error "npm ERR! code CERT_HAS_EXPIRED".

Created2024-01-01|Updated2024-05-29
|Post Views:

想要快速建立一个个人主页。

几年前也曾有过类似想法，想着不能任由自己自闭下去了。但当时有工作压力，社会压力，加上高内耗性格放大了这一切负面影响，因此并没能在office time外养成任何持续的余暇活动，包括写博客这一愿望也并未能实际展开。某天找回闲置的github账号，才发现近十年前曾在同事鼓动下新建了个jekyll repo，但内容空无一物，各种配置包括github pages也没搞好。

Google一下，这几年来主流方向似乎没什么变化，依旧是诸如WordPress，Google Site，Hexo，Typecho之类。后头两个并不是新鲜工具，这还是从mygithub团队项目里瞥见它两的痕迹才意识到的。由于我实在想既快速又可编码，并不想考虑必须上vps的方案，最后决定就是Hexo了。

Hexo由Nodejs开发。我虽然对nodejs不熟，但好歹使用没问题，本地环境也都还健在。没想到第一步安装npm包就被磕绊一脚。

上面这个命令报错“npm ERR! code CERT_HAS_EXPIRED”。我脑子没反应过来，第一反应是npm怎么会存在过期的问题呢？大概是我的npm版本太低了（v6的确也相当老旧了）。于是遇事不决，版本升级（其实没用）。

但恼人的是，npm install npm -g执行成功也依旧版本未更新（还不如不成功）。奇了怪了？此时我竟没意识到，npm是包管理器，而包管理器都是依附于特定语言的。npm依附于nodejs，因此nodejs版本直接决定了npm版本的更新范围。我忙活好一阵，才想起本地在用nvm管理nodejs版本，于是立刻用它安装了最新版nodejs。然而，最新版本v21安装貌似成功，却实则不能使用。原来v21并不支持MacOS版本10.12（可恶的是安装过程并无报错！）。于是又改为较老一些的v16，终于可用。其中nodejs v16对应着npm v8。

然而经过一波三折，不幸的是nodejs+npm版本更新对报错没用。其实根本原因与版本无关，而在于npm源失效了。啊~~~我脑子实在愚钝，怎么没想到npm作为包管理器肯定得配置国内源才能实用！？

总结，“npm ERR! code CERT_HAS_EXPIRED”根本原因是npm的国内源改hostname域名了。
